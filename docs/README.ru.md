# Claude Usage Monitor

[English](../README.md) · [简体中文](README.zh-CN.md) · [日本語](README.ja.md) · [한국어](README.ko.md) · [Deutsch](README.de.md) · [Français](README.fr.md) · [Português](README.pt.md) · **Русский**

Нативное приложение для строки меню macOS, которое в реальном времени отслеживает официальные лимиты использования и оценочные расходы **Claude Code**.

> Интерфейс приложения автоматически подстраивается под язык системы macOS — английский, китайский (упрощённый), японский, корейский, немецкий, французский, португальский, русский — а в остальных случаях откатывается на английский.

## Требования

- macOS 13+
- Установленный CLI **Claude Code** с выполненным входом (приложение читает официальные лимиты через `claude -p '/usage'`)

## Установка

1. Скачайте последний `Claude Usage Monitor.zip` из раздела [Releases](../../../releases) и распакуйте.
2. Перетащите `Claude Usage Monitor.app` в «Программы».
3. Если macOS блокирует первый запуск: правый клик по приложению → «Открыть»; или откройте «Системные настройки → Конфиденциальность и безопасность» и нажмите «Всё равно открыть».

> Приложение подписано локально в режиме ad-hoc и не заверено Apple, поэтому при первом запуске требуется ручное разрешение.

## Использование

После запуска оно живёт только в **строке меню** — без значка в Dock.

- В заголовке строки меню отображается текущий **пиковый процент использования лимита (%)**; чем ближе к лимиту, тем краснее.
- **Нажмите на значок**, чтобы открыть панель:
  - **Лимиты использования · Официально** — процент использования + время сброса для текущей сессии, текущей недели (все модели) и текущей недели (Fable). Полностью совпадает с `/usage` в Claude Code.
  - **Окна активности** — число запросов, сессий и доля длинного контекста за последние 24 ч / 7 дней.
  - **Основные источники** — skills / plugins / MCP / subagents, внёсшие наибольший вклад за последние 7 дней.
  - **Оценка расходов · Локальные логи** — расходы за сегодня / последние 7 дней / всё время, оценённые из `~/.claude/projects/**/*.jsonl` по публичным тарифам, с гистограммой за 14 дней.

Частота обновления: официальные лимиты — каждые 5 минут, локальные расходы — каждые 60 секунд.

### Запуск при входе

Системные настройки → Основные → Объекты входа → добавьте `Claude Usage Monitor.app`.

## Настройка тарифов

Расходы оцениваются по публичным тарифам Anthropic (Fable/Mythos временно тарифицируются по уровню Opus). Чтобы переопределить, создайте
`~/.claude/claude-usage-monitor/pricing.json`, единицы измерения — «USD за миллион токенов»:

```json
{
  "fable": { "input": 15, "output": 75, "cacheWrite5m": 18.75, "cacheWrite1h": 30, "cacheRead": 1.5 }
}
```

Ключи сопоставляются по подстроке идентификатора модели (например, `opus`, `sonnet`, `haiku`, `fable`). Модели не от Anthropic без тарифа считаются с нулевой стоимостью, но их токены всё равно учитываются.

## Сборка из исходников

```bash
./build.sh
open "dist/Claude Usage Monitor.app"
```

Требуется Swift 6 (инструменты командной строки Xcode). Результат появляется в `dist/` (не отслеживается системой контроля версий).
